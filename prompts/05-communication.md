# Communication & docs

## PR description

```
Draft a PR description for:

Title: <short>
What: <bullets>
Why: <motivation>
How: <high-level approach>
Risk / rollback: <notes>
Testing: <commands run, manual checks>

Tone: concise, reviewer-friendly, no hype.
```

## Commit message(s)

```
Suggest 1–3 commits splitting this work logically.

For each commit: one-line subject (imperative), body with context if non-obvious.
Follow conventional commits if this repo uses them; otherwise plain style.
```

## Changelog entry

```
Audience: <end users | developers>

Summarize user-visible changes for version <x.y.z>.
Omit internal refactors unless they affect behavior or upgrades.
```

## Weekly status update

```
Write a short status update:

Done: <bullets>
Next: <bullets>
Blocked: <bullets or “none”>
Asks: <decisions or help needed>

Tone: factual, scannable.
```
